# Explanation

**Understanding-oriented** documentation: the concepts, architecture and rationale behind ProcureFlow.
Explanation answers *"How does this fit together, and why is it like this?"* — it gives the reader a
mental model, as opposed to steps ([`../guides/`](../guides/)) or look-up facts ([`../reference/`](../reference/)).

## In this folder

- [`documentation.md`](./documentation.md) — "How we document": the Diátaxis split, when an ADR is
  required, and the authoring conventions.

Architecture and domain concepts (the request state machine, the budget invariant, module boundaries)
are explained here as the corresponding features land. The project constitution
[`../../CLAUDE.md`](../../CLAUDE.md) holds the binding architecture conventions; explanation docs
expand on the *why*.
