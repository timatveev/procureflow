# Guides (how-to)

**Task-oriented** documentation: step-by-step recipes that get a competent reader to a specific goal.
A guide answers *"How do I do X?"* — it assumes you know *why* and just need the steps.

Keep guides short, imperative, and outcome-focused. For background and rationale, link to
[`../explanation/`](../explanation/); for exhaustive look-up detail, link to [`../reference/`](../reference/).

## In this folder

- [`writing-an-adr.md`](./writing-an-adr.md) — record an architecture decision.

More guides are added as features land (e.g. running migrations, adding a module).
